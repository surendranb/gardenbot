# Hello World: The Origins of GardenOS

I wanted to build something that actually watches my plants — not just logs numbers, but looks at them, checks the sensors, and tells me if something's off. That's GardenOS. Three pots on a desk in Chennai, monitored by an Arduino, a webcam, and some Python scripts.

<!-- more -->

## 🛠 The hardware

Kept it simple:

* An old MacBook Air running the scripts and OpenClaw
* An Arduino as the serial bridge for the sensor array
* A DHT11 for temperature and humidity
* A lux sensor for light
* Three capacitive moisture sensors (p1: Nickels, p2: Mint, p3: Pothos)
* A webcam mounted above the desk for top-down shots

## 🧠 How it works

The system isn't one big program — it's a pipeline of independent scripts that each do one thing.

1. `warden.py`, `vision.py`, and `weather_scout.py` run on a schedule via cron/launchd. Plain Python, no framework.
2. `warden.py` reads the Arduino over serial — temp, humidity, light, soil moisture — and writes it all to `telemetry.csv`, `metrics.csv`, `current_snapshot.json`, and `warden_state.json`.
3. `vision.py` grabs a frame from the webcam, sends it to Gemma 3 on Google AI Studio. Gemma describes what it sees — leaf color, posture, soil surface. Output goes to `vision_observation.json` and `vision_observation.md`.
4. `weather_scout.py` pulls current Chennai weather from OpenWeatherMap. This is the outdoor macro-context — useful because what's happening outside is almost completely decoupled from what's happening on the desk.
5. `prep_observer_context.py` (the SILICA context layer) takes all those data files, merges them with the world model and plant config, and produces `observer_context.md`. This is where raw numbers turn into facts the LLM can reason about.
6. OpenClaw's Warden agent reads the context and calls an LLM. It cross-checks sensor data against visual evidence, looks at trends, and flags anything that needs attention.
7. The report goes to `vision_ledger.md` and gets posted to Slack.
8. `sync.sh` builds the MkDocs site, commits everything to GitHub, and pushes to Pages.

The key thing is decoupling. Each script runs on its own. If the LLM goes down, sensors still log. If the webcam is unplugged, telemetry still records. If the weather API is flaky, everything else keeps going.

## 🌿 Final thoughts

It's three pots on a desk. But it's a monitored biome, and that's the fun part.
